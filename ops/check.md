# Context Checklist — SMB Program Site Build
> Use before, during, and after every AI session.

## Before Session
- [ ] Start a new session (fresh context)
- [ ] Paste/upload `ops/rules.md`
- [ ] Paste/upload `ops/canon.md`
- [ ] Paste/upload `ops/sum.md`
- [ ] State today's task clearly (one task type)
- [ ] Provide only relevant source files for this task
- [ ] Choose task type: Writing / Coding / Research / Strategy / Ops

## During Session
- [ ] Re-anchor rules + canon every 10–15 turns
- [ ] Watch drift signals (format, scope, omissions, style)
- [ ] Keep session to one task type when possible
- [ ] Save good outputs immediately into repo files
- [ ] If quality drops, stop and start a new session

## After Session
- [ ] Save all outputs to project folder/repo
- [ ] Update `ops/sum.md` (<100 words)
- [ ] Update `ops/log.md`
- [ ] Update `ops/canon.md` (Decision Log / Status)
- [ ] Note reusable prompts in `prompts/tasks/`
- [ ] Version/commit changes
