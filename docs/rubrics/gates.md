# Gates Rubric

## Objective

Evaluate whether a learner has demonstrated a competency gate strongly enough to advance.

## Required Deliverables for Gate Review

- Week/module deliverable bundle
- Rubric self-assessment
- Short verification note explaining evidence for gate competency

## Rubric

| Criterion | Weight | What "Meets" Looks Like |
| --- | ---: | --- |
| Competency demonstration | 35% | Learner shows the target skill through observable output |
| Artifact completeness | 25% | Required artifacts are complete, coherent, and aligned to scope |
| Reproducibility and verification | 20% | Reviewer can validate claims using submitted evidence |
| Decision quality | 10% | Prioritization and tradeoffs are explicit and defensible |
| Professional delivery | 10% | Communication is clear, concise, and reviewer-ready |

## Scoring Guidance

- 4 = Strong independent performance
- 3 = Ready with minor revisions
- 2 = Not yet consistent; major gaps remain
- 1 = Not demonstrated

## Pass Criteria

- Weighted score >= 80/100
- No score below 3 in `Competency demonstration` or `Reproducibility and verification`
- Required artifacts and verification note submitted

## Ship Checklist (Reviewer)

- [ ] Competency target is clearly identified
- [ ] Evidence directly maps to gate criteria
- [ ] Pass/fail decision is logged
- [ ] Required remediation steps are listed when failing
